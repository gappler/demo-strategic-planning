OKR Tool — Layout Approaches

Two approaches for organizing scenario-based OKRs with cascading goals

A

Tabbed Scenario View

Top-level tabs for Base / Upside / Downside scenarios. Below the tabs, a single scrollable page with all four functional areas (Marketing, Product, Ops, Finance) as collapsible accordion sections. Each section shows objectives → key results → resource tags.

Pros

  • Simple mental model — one scenario at a time
  • Full vertical space for OKR detail
  • Easy to scan all functions in one scroll

Cons

  • Can't compare scenarios side-by-side
  • Switching tabs loses scroll position
B

Dashboard with Scenario Switcher

A persistent sidebar shows the scenario selector, strategic context summary, and aggregate resource totals. The main area is a card grid — one card per objective, color-coded by function, showing KRs and resource tags. Switching scenarios animates cards in/out as objectives change.

Pros

  • Scenario context always visible
  • Card layout lets you see all objectives at a glance
  • Animated transitions make trade-offs tangible
  • Resource summary updates in real-time in sidebar

Cons

  • Less vertical space for KR detail per card
  • More complex to build
